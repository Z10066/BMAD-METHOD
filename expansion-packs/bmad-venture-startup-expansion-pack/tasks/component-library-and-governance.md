# Component Library & Governance

## Purpose

建设组件库与规范、评审与发布流程（Figma/代码一体）。

## Inputs

- 设计系统基础层/Token
- templates/component-spec-tmpl.yaml
- templates/figma-contribution-guide-tmpl.yaml

## Outputs

- 组件规格/示例与贡献/发布流程。

## Steps

1. 确定核心组件优先级
2. 定义属性/交互/状态/可达性
3. 建立Figma与代码映射与Story
4. 设评审/版本与破坏性变更策略
5. 度量复用与满意度
