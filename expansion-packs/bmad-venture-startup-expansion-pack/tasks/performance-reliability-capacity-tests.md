# Performance, Reliability & Capacity Tests

## Purpose

形成性能/可靠性/容量基线与压测计划。

## Inputs

- 流量/依赖/SLO/历史瓶颈
- templates/perf-test-plan-tmpl.yaml
- checklists/performance-test-readiness.md

## Outputs

- 性能压测报告与建议。

## Steps

1. 确定目标与场景：负载/峰值/疲劳/灾备
2. 指标：延迟/吞吐/错误/资源/成本
3. 稳定性策略与配额/限流/队列
4. 回归与趋势/容量模型联动
5. 进入发布门质量评审
